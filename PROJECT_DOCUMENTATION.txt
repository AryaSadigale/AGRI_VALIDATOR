## Project Overview

**Agricultural Recommendation Validator** is an advanced AI-powered system designed to validate crop recommendations using a multi-layered validation approach. The system combines machine learning models, expert rule-based systems, and large language models (LLMs) to provide reliable agricultural risk assessments.

### Purpose
- Validate agricultural crop recommendations before they reach farmers
- Prevent incorrect or risky agricultural advice
- Provide confidence scores and reliability metrics
- Generate AI-powered advisory text for farmers
- Enable batch processing of large datasets
- Maintain audit logs for compliance and tracking

---

## What This Project Does

### Core Functionality

1. **Single Prediction Validation**
   - Accepts agricultural input data (State, District, Crop, Season, Area, Production, Yield)
   - Predicts crop risk level (Low, Medium, High)
   - Validates prediction using multiple layers
   - Generates AI-powered advisory text
   - Returns comprehensive metrics and confidence scores

2. **Batch Dataset Validation**
   - Processes CSV files containing multiple agricultural records
   - Validates each record through the multi-layer system
   - Generates decision actions (AUTO_FORWARD, SEND_TO_OFFICER, BLOCK_AI_OUTPUT)
   - Creates downloadable validation reports
   - Updates audit logs with processing statistics

3. **Dashboard Analytics**
   - Real-time statistics on validation results
   - Tracks approved, review required, and rejected recommendations
   - Displays last run time and total records processed
   - Provides downloadable reports

## How It Works

### Multi-Layered Validation System

The system uses **three independent validation layers** that work together to ensure accuracy:

#### Layer 1: AI/ML Model (XGBoost)
- **Technology**: Calibrated XGBoost classifier
- **Training**: Trained on historical agricultural data
- **Output**: Risk prediction (Low/Medium/High) with probability scores
- **Metric**: Prediction Confidence Score (PCS)

**Process:**
1. Encodes categorical features (State, District, Crop, Season) using pre-trained label encoders
2. Combines with numerical features (Area, Production, Yield)
3. Feeds into calibrated XGBoost model
4. Returns probability distribution across risk classes
5. Selects highest probability as prediction

#### Layer 2: Expert Rule-Based System
- **Technology**: Domain expert knowledge encoded as rules
- **Logic**: Crop-specific yield thresholds
- **Output**: Risk assessment based on agricultural expertise

**Rules:**
```
Rice:
  - Yield < 2200 kg/ha → High Risk
  - Yield 2200-3000 kg/ha → Medium Risk
  - Yield > 3000 kg/ha → Low Risk

Wheat:
  - Yield < 2500 kg/ha → High Risk
  - Yield 2500-3500 kg/ha → Medium Risk
  - Yield > 3500 kg/ha → Low Risk

Other Crops:
  - Yield < 2000 kg/ha → High Risk
  - Yield 2000-3000 kg/ha → Medium Risk
  - Yield > 3000 kg/ha → Low Risk
```

#### Layer 3: LLM Validation (Gemini)
- **Technology**: Google Gemini AI
- **Purpose**: Generate human-readable advisory and validate consistency
- **Output**: Advisory text + validation metrics

**Process:**
1. Generates agricultural advisory based on input data and AI risk prediction
2. Validates advisory text for:
   - Rule compliance
   - Text similarity with expected patterns
   - Data consistency
3. Returns LLM Trust Score (LTS) and validation status

## Validation Methodology

### Step-by-Step Validation Process

#### 1. Data Input
```
Input: {
  State: "Maharashtra",
  District: "Pune",
  Crop: "Rice",
  Season: "Kharif",
  Area: 10.5,
  Production: 28.5,
  Yield: 2714.29
}
```

#### 2. AI Model Prediction
- Encodes categorical features
- Runs through XGBoost model
- Gets probability distribution: [0.15, 0.70, 0.15] for [Low, Medium, High]
- Prediction: Medium Risk
- PCS (Prediction Confidence Score): 0.70 (70%)

#### 3. Expert System Validation
- Checks crop type: Rice
- Checks yield: 2714.29 kg/ha
- Applies rule: 2200 < 2714.29 < 3000 → Medium Risk
- Expert Risk: Medium

#### 4. Agreement Calculation

**Expert Agreement Score (EAS):**
```
Risk Mapping: Low=1, Medium=2, High=3
AI Value: 2 (Medium)
Expert Value: 2 (Medium)
EAS = 1 - (|2 - 2| / 2) = 1.0 (100% agreement)
```

**Risk Deviation Index (RDI):**
```
RDI = |2 - 2| / 2 = 0.0 (no deviation)
```

#### 5. Trust Calculation

**Trust Reliability Index (TRI):**
```
TRI = (PCS × 0.6 + EAS × 0.4) × 100
TRI = (0.70 × 0.6 + 1.0 × 0.4) × 100
TRI = (0.42 + 0.40) × 100 = 82%
```

#### 6. Validation Status

**Decision Logic:**
```
TRI ≥ 80% → APPROVED
60% ≤ TRI < 80% → REVIEW REQUIRED
TRI < 60% → REJECTED
```

**Confidence Band:**
```
TRI ≥ 90% → Very High Reliability
TRI ≥ 80% → High Reliability
TRI ≥ 65% → Moderate Reliability
TRI < 65% → Low Reliability
```

#### 7. LLM Advisory Generation
- Generates context-aware advisory text
- Validates advisory for consistency
- Calculates LLM-specific metrics:
  - **RCS** (Rule Compliance Score)
  - **ASS** (Advisory Similarity Score)
  - **DCS** (Data Consistency Score)
  - **LTS** (LLM Trust Score)

## Metrics and Scoring

### Prediction Confidence Score (PCS)
**Formula**: `max(probability_distribution)`

**Meaning**: Confidence of the AI model in its prediction

**Range**: 0.0 to 1.0 (0% to 100%)

**Example**: If probabilities are [0.15, 0.70, 0.15], PCS = 0.70

---

### Expert Agreement Score (EAS)
**Formula**: `1 - (|AI_value - Expert_value| / 2)`

**Meaning**: Agreement between AI and expert system

**Range**: 0.0 to 1.0 (0% to 100%)

**Examples**:
- Both predict Medium: EAS = 1.0 (perfect agreement)
- AI predicts Medium, Expert predicts High: EAS = 0.5 (50% agreement)
- AI predicts Low, Expert predicts High: EAS = 0.0 (no agreement)

---

### Risk Deviation Index (RDI)
**Formula**: `|AI_value - Expert_value| / 2`

**Meaning**: Deviation between AI and expert predictions

**Range**: 0.0 to 1.0

**Interpretation**:
- 0.0 = No deviation (perfect agreement)
- 0.5 = One level difference
- 1.0 = Maximum deviation (opposite predictions)

---

### Trust Reliability Index (TRI)
**Formula**: `(PCS × 0.6 + EAS × 0.4) × 100`

**Meaning**: Overall trust and reliability score

**Weights**:
- PCS: 60% (AI model confidence)
- EAS: 40% (Expert agreement)

**Range**: 0 to 100

**Decision Thresholds**:
- TRI ≥ 80: APPROVED
- 60 ≤ TRI < 80: REVIEW REQUIRED
- TRI < 60: REJECTED

**Example Calculation**:
```
PCS = 0.85 (85% AI confidence)
EAS = 1.0 (100% expert agreement)
TRI = (0.85 × 0.6 + 1.0 × 0.4) × 100
TRI = (0.51 + 0.40) × 100 = 91%
Status: APPROVED (Very High Reliability)
```

---

### LLM Validation Metrics

**RCS (Rule Compliance Score)**:
- Measures adherence to agricultural rules
- Range: 0.0 to 1.0

**ASS (Advisory Similarity Score)**:
- Measures similarity to expected advisory patterns
- Range: 0.0 to 1.0

**DCS (Data Consistency Score)**:
- Measures consistency between input data and advisory
- Range: 0.0 to 1.0

**LTS (LLM Trust Score)**:
- Overall LLM reliability
- Formula: `(RCS + ASS + DCS) / 3 × 100`
- Range: 0 to 100
